# 기능정의서

## 1. 서비스 개요

**서비스명:** 하루AI (HaruAI)
**목적:** AI 대화를 통해 일기 작성의 진입 장벽을 낮추고, 누적된 일기 데이터를 기반으로 자기 인사이트를 제공

**핵심 사용자 페인포인트:**
- 쓰는 것 자체가 귀찮음
- 뭘 써야 할지 모르겠음
- 쓰다가 며칠 빼먹으면 포기
- 나중에 돌아보고 싶은데 방법이 없음

## 2. Phase 1 — MVP 기능

### F1. 대화형 일기 작성

**설명:** AI가 질문을 통해 사용자의 하루를 끌어내고, 대화 내용을 바탕으로 일기를 자동 생성한다.

**대화 플로우:**
```
1. 사용자가 일기 세션 시작
2. AI가 첫 질문 ("오늘 하루 어땠어?")
3. 사용자 짧은 답변
4. AI가 맥락에 맞는 후속 질문 (3~5회 반복)
5. 충분한 내용이 모이면 AI가 일기 초안 생성
6. 사용자 확인/수정 후 저장
```

**AI 질문 전략:**
- 첫 질문: 넓고 가볍게 ("오늘 하루 어땠어?", "오늘 특별한 일 있었어?")
- 후속 질문: 답변에서 키워드 추출 → 구체화 ("그 회의에서 어떤 얘기 나왔어?")
- 감정 탐색: 감정 표현 감지 시 → 더 깊이 ("그때 기분이 어땠어?")
- 마무리: "오늘 하루를 한마디로 하면?" 또는 자연스럽게 종료 판단

**일기 생성 규칙:**
- 1인칭 시점, 일기체
- 대화에서 나온 사실만 포함 (AI가 내용을 지어내지 않음)
- 시간 순서대로 정리
- 감정 표현은 사용자의 원래 표현 유지
- 길이: 200~500자 내외

**입력:** 텍스트 채팅
**출력:** 자동 생성된 일기 + 감정 태그

### F2. 감정 분석 및 태깅

**설명:** 대화 내용과 생성된 일기를 분석하여 감정을 자동 태깅한다.

**감정 카테고리:**
| 감정 | 분류 | 설명 | 예시 |
|------|------|------|------|
| joy | 긍정 | 기쁨, 즐거움 | 맛집 발견, 좋은 소식 |
| proud | 긍정 | 뿌듯함, 성취감 | 프로젝트 완료, 칭찬 |
| excited | 긍정 | 설렘, 기대 | 여행 계획, 새 시작 |
| grateful | 긍정 | 감사, 고마움 | 도움 받음, 행운 |
| relieved | 긍정 | 안도, 해방감 | 발표 끝, 걱정 해소 |
| content | 긍정 | 만족, 충족감 | 맛있는 식사, 좋은 휴식 |
| calm | 중립 | 평온, 편안 | 평범한 하루 |
| bored | 중립 | 지루함, 무료함 | 할 일 없는 날 |
| nostalgic | 중립 | 그리움, 추억 | 옛 친구 생각, 고향 |
| tired | 부정 | 피곤, 지침 | 야근, 과로 |
| sadness | 부정 | 슬픔, 우울 | 이별, 실패 |
| anger | 부정 | 화남, 짜증 | 갈등, 부당한 일 |
| anxiety | 부정 | 불안, 걱정 | 면접, 마감 |
| lonely | 부정 | 외로움, 고독 | 혼자인 날, 소외감 |
| frustrated | 부정 | 답답함, 막막함 | 버그, 막힌 상황 |

**분석 결과 JSON:**
```json
{
  "primary_emotion": "anger",
  "secondary_emotion": "anxiety",
  "intensity": 7,          // 1~10
  "keywords": ["야근", "버그", "마감"],
  "summary": "업무 과부하로 인한 스트레스"
}
```

### F3. 일기 저장 및 조회

**저장 구조:**
- PostgreSQL: 일기 원문, 날짜, 감정 태그, 대화 내역
- ChromaDB: 일기 텍스트 임베딩 (시맨틱 검색용)

**조회 기능:**
- 날짜별 조회 (특정일, 기간)
- 감정별 필터링 ("화난 날들만 보여줘")
- 최근 일기 목록

## 3. Phase 2 — RAG 인사이트

### F4. 과거 일기 시맨틱 검색

**설명:** 자연어 질문으로 과거 일기를 검색한다. 키워드 매칭이 아닌 의미 기반 검색.

**질문 유형별 처리:**
| 질문 유형 | 예시 | 처리 방식 |
|-----------|------|-----------|
| 시간 기반 | "지난주 뭐 했지?" | RDB 날짜 쿼리 |
| 감정 기반 | "내가 제일 행복했던 날" | RDB 감정 태그 + RAG |
| 주제 기반 | "회사에서 힘들었던 적" | ChromaDB 시맨틱 검색 |
| 복합 | "지난달에 야근 많이 했지?" | RDB + ChromaDB 결합 |

### F5. 맥락 기반 후속 질문

**설명:** 이전 일기 내용을 참고하여 오늘의 질문에 반영한다.

**예시:**
- 어제 일기: "프로젝트 발표 준비 때문에 긴장됐다"
- 오늘 AI 질문: "어제 긴장된다고 했던 발표, 잘 끝났어?"

**구현 방식:**
- 일기 세션 시작 시 최근 3일 일기를 컨텍스트로 주입
- LLM이 연속성 있는 질문 생성

### F6. 주간/월간 리포트

**설명:** 누적된 일기 데이터를 분석하여 정기 리포트를 생성한다.

**리포트 내용:**
- 감정 분포 (이번 주 기쁨 40%, 불안 30%, ...)
- 주요 이벤트 요약
- 반복 패턴 감지 ("수요일마다 스트레스 높음")
- 감정 변화 추이
- AI 한마디 (격려, 조언)

### F7. 패턴 분석

**설명:** 장기간 데이터에서 사용자가 인식하지 못한 패턴을 발견한다.

**분석 항목:**
- 감정 트리거 ("야근 언급 시 anger 태그 80%")
- 요일별 감정 패턴
- 반복되는 고민/주제
- 긍정/부정 추세 변화

## 4. Phase 3 — 음성 입력

### F8. 음성 일기

**설명:** 음성으로 대화하여 일기를 작성한다.

**플로우:**
```
1. 음성 녹음 시작
2. STT로 텍스트 변환
3. AI 질문 (텍스트 or TTS)
4. 음성 답변 → STT → 반복
5. 일기 생성
```

**기술:** Whisper API (OpenAI) 또는 Google Speech-to-Text

## 5. 비기능 요구사항

- **프라이버시:** 일기 데이터는 사용자 본인만 접근 가능
- **응답 시간:** 대화 응답 3초 이내, 일기 생성 5초 이내
- **데이터 보존:** 사용자 요청 시 전체 데이터 삭제 가능
- **일기 내보내기:** 텍스트/JSON 형태로 다운로드 가능
